# CI/CD

Structured reference for agents and contributors. Continuous integration on GitHub Actions; deployment is manual or provider-specific until chosen.

---

## Platform

| Field           | Value                                                                                                                          |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| **Choice**      | GitHub Actions                                                                                                                 |
| **Role**        | Run lint, typecheck, and tests on push and PR                                                                                  |
| **Rationale**   | Standard for GitHub-hosted repos; no extra service. Matches lefthook locally — CI is the remote enforcement of the same gates. |
| **Conventions** | Workflow file: `.github/workflows/ci.yml`. Cache pnpm store. Use `DATABASE_URL=file::memory:?cache=shared` in CI tests.        |
| **References**  | https://docs.github.com/en/actions                                                                                             |

**Deployment:** not automated in v1. Deploy via Docker on VPS or `fly deploy` / Railway CLI when operator chooses. Add `deploy.yml` later if needed.

---

## CI workflow

Trigger: `push` to `main`, all `pull_request`.

Jobs:

### `check` (single job)

| Step         | Command                                                                     |
| ------------ | --------------------------------------------------------------------------- |
| Checkout     | `actions/checkout@v4`                                                       |
| Node         | `actions/setup-node@v4` — Node 22 LTS                                       |
| pnpm         | `pnpm/action-setup@v4` — version from `package.json` `packageManager` field |
| Install      | `pnpm install --frozen-lockfile`                                            |
| Lint         | `pnpm lint`                                                                 |
| Format check | `pnpm format:check`                                                         |
| Typecheck    | `pnpm typecheck`                                                            |
| Test         | `pnpm test`                                                                 |
| Build        | `pnpm build`                                                                |

Environment for test step:

```yaml
env:
  DATABASE_URL: "file::memory:?cache=shared"
  SESSION_SECRET: "ci-test-secret-not-for-production"
```

The workflow also runs `pnpm build` as a final step and exposes a `workflow_call` trigger so other workflows (e.g. image publishing) can reuse it. No Docker image build runs inside this `check` job.

---

## Workflow file sketch

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
  workflow_call:

jobs:
  check:
    runs-on: ubuntu-latest
    env:
      DATABASE_URL: "file::memory:?cache=shared"
      SESSION_SECRET: "ci-test-secret-not-for-production"
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm lint
      - run: pnpm format:check
      - run: pnpm typecheck
      - run: pnpm test
      - run: pnpm build
```

Implement when `package.json` exists (Phase 0).

## Local parity

| Gate          | Local (lefthook) | CI           |
| ------------- | ---------------- | ------------ |
| Lint staged   | pre-commit       | full project |
| Format staged | pre-commit       | full project |
| Typecheck     | pre-commit       | full project |
| Test          | pre-push         | full suite   |

CI runs full-project lint/format; lefthook scopes to staged files for speed.

---

## Branch protection (recommended)

On GitHub repo settings:

- Require PR before merge to `main`
- Require `check` job to pass
- Require conventional commits (optional; commitlint is local on commit-msg hook)
